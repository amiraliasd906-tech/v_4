# جستجوی هوشمند دیوار — اپلیکیشن اندروید (کاملاً مستقل، بدون سرور)

Kotlin + Jetpack Compose، معماری Clean Architecture (Data / Domain / Presentation) + MVVM، Material Design 3.

## این نسخه چه فرقی با نسخه قبلی دارد؟

نسخه‌های قبلی به یک بک‌اند Python جدا (روی کامپیوتر) و یک افزونه مرورگر Chrome
نیاز داشتند. **این نسخه کاملاً مستقل است** — همه‌چیز داخل خود اپ اجرا می‌شود:

| قبل | حالا |
|---|---|
| سرور FastAPI + PostgreSQL | دیتابیس محلی روی گوشی (Room / SQLite) |
| افزونه Chrome روی کامپیوتر | یک تب مرورگر **داخل خود اپ** (WebView) |
| فیلترها/AI روی سرور (Python) | همان منطق، پورت‌شده به Kotlin، مستقیم روی گوشی |
| نوتیف از طریق Firebase (FCM) | نوتیف محلی مستقیم از خود اپ |

فقط یک اپ نصب می‌کنید — نه کامپیوتر لازم است، نه سرور، نه افزونه.

## فلوی استفاده

1. **جستجوی جدید بسازید** (فرم فیلترها → فیلترهای دائمی → ذخیره)
2. در «جستجوهای من»، روی آیکون 🧭 «باز کردن و رصد» بزنید
3. یک تب مرورگر داخل خود اپ باز می‌شود که صفحه جستجوی دیوار را نشان می‌دهد
4. همان‌طور که در دیوار می‌گردید (لیست یا صفحه یک آگهی)، اپ به‌صورت خودکار و
   passively (بدون هیچ درخواست شبکه‌ای اضافه) متن صفحه را می‌خواند، فیلتر
   می‌کند، و نتایج را ذخیره/اعلان می‌دهد
5. نتایج در تب «نتایج» و تاریخچه در تب «تاریخچه» قابل مشاهده است

## معماری

```
presentation/   Compose screens + ViewModels (MVVM)
domain/         Use cases + repository interfaces + domain models
data/
  local/        Room database, entities, DAOs, mapperها
  filters/      فیلتر شماره مسدود، کلمات ممنوعه، تشخیص مالک با AI (Kotlin)
  webview/      استخراج آگهی از WebView + سرویس ingestion
  repository/   پیاده‌سازی repositoryها روی Room
di/             ماژول‌های Hilt (Database, Repository)
notification/   نوتیفیکیشن محلی (بدون FCM/سرور)
```

## فیلترها (همه روی گوشی، بدون سرور)

- **فیلترهای دائمی (شماره مسدود)**: هم فیلد رسمی تماس و هم شماره‌های نوشته‌شده
  مستقیم در متن آگهی چک می‌شوند (`data/filters/PhoneFilter.kt`)
- **کلمات ممنوعه**: اگر «مشاور» یا «املاک» در توضیحات باشد، آگهی همیشه مخفی
  می‌شود — بدون استثنا (`data/filters/ForbiddenKeywords.kt`)
- **تشخیص مالک با AI**: یک نسخه heuristic آفلاین (پیش‌فرض) + یک نسخه که مستقیم
  از گوشی به Claude API وصل می‌شود اگر در تنظیمات کلید API خودتان را وارد کنید
  (`data/filters/OwnerDetector.kt`)

## راه‌اندازی

1. Android Studio را باز کنید ← Open ← این پوشه را انتخاب کنید
2. صبر کنید Gradle Sync تمام شود
3. گوشی را با USB وصل کنید (USB Debugging فعال باشد) و Run ▶️ بزنید

نیازی به تنظیم هیچ آدرس سرور یا فایل پیکربندی نیست — همه‌چیز از همان لحظه
اول کار می‌کند.

### فعال‌سازی تحلیل هوشمند متن با AI (اختیاری)

در تب «تنظیمات»، کلید API آنتروپیک (Anthropic) خودتان را وارد کنید. بدون این
کلید، تشخیص مالک با روش قانون‌محور آفلاین (heuristic) انجام می‌شود که همیشه
در دسترس است.

## محدودیت‌های شفاف

- چون استخراج داده فقط زمانی اتفاق می‌افتد که **خودتان** صفحه دیوار را در تب
  داخلی اپ باز کرده باشید، این اپ به‌صورت خودکار در پس‌زمینه (وقتی بسته است)
  آگهی جدید پیدا نمی‌کند — باید هر بار خودتان صفحه جستجو را باز کنید.
- شماره تماس و توضیحات کامل آگهی معمولاً فقط در **صفحه خود آگهی** نمایش داده
  می‌شوند، نه در لیست نتایج — پس فیلتر شماره/AI تا وقتی آن آگهی را باز نکنید،
  اطلاعات کامل ندارد (دقیقاً مثل محدودیت افزونه مرورگر نسخه قبلی).
- سلکتورهای استخراج در `JsExtractionScripts.kt` بر پایه ساختار عمومی HTML
  حدس زده شده‌اند (بدون دسترسی مستقیم به صفحه واقعی دیوار در این محیط توسعه)
  و ممکن است نیاز به تنظیم جزئی داشته باشند.
